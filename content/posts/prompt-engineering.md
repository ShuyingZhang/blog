# LLM提示词工程入门指南

提示词工程（Prompt Engineering）是与大型语言模型（LLM）有效沟通的关键技能。这篇博客将介绍一些实用的技巧和策略。

## 什么是提示词工程

提示词工程是设计和优化输入到AI模型的文本提示，以获得更好、更准确的输出的过程。

## 基本原则

### 1. 明确具体

```python
# ❌ 不好的提示
"写一些关于AI的内容"

# ✅ 好的提示
"写一篇500字的文章，介绍机器学习在医疗诊断中的应用，包含至少三个具体案例"
```

### 2. 提供上下文

给模型提供足够的背景信息，帮助它理解任务需求。

### 3. 使用分隔符

使用```或其他分隔符来区分不同部分的内容。

## 实用技巧

### 零样本提示（Zero-shot）

直接给出任务，不提供示例：

> "将以下文本分类为正面、中性或负面：'这款产品非常好用'"

### 少样本提示（Few-shot）

提供几个示例来引导模型：

> "任务：情感分析
> 示例：
> '太棒了！' → 正面
> '很失望' → 负面
> '还行' → 中性
> '服务态度很好' → ?"

### 思维链提示（Chain-of-Thought）

要求模型展示推理过程：

> "请一步步思考，计算以下问题：..."
> "首先分析问题，然后给出解决步骤，最后得出答案。"

## 进阶技巧

### 角色扮演

```markdown
你是一位资深的AI研究专家，拥有10年机器学习研究经验。
请用专业但易懂的语言解释什么是Transformer架构。
```

### 结构化输出

```markdown
请按以下JSON格式输出结果：
{
  "summary": "文章摘要",
  "keywords": ["关键词1", "关键词2"],
  "sentiment": "正面/中性/负面"
}
```

### 迭代优化

如果第一次结果不满意，可以：
- 调整提示词的表述
- 添加更多约束条件
- 要求模型提供多个选项

## 常见错误

1. **过于模糊** - 没有明确表达需求
2. **信息不足** - 缺少必要的上下文
3. **指令冲突** - 自相矛盾的要求
4. **期望过高** - 要求超出模型能力范围

## 总结

提示词工程是一门艺术和科学的结合。通过不断实践和优化，我们可以更好地利用AI工具，提高工作效率。

记住：好的提示词 = 清晰的目标 + 充分的上下文 + 适当的约束
